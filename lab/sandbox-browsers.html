🧪 Sandbox Browsers

Sandbox-браузеры — это изолированные среды исполнения веб-контента, в которых каждая вкладка, процесс или страница работает в ограниченной "песочнице".

🧠 Что такое Sandbox в браузерах

Sandbox (песочница) — это механизм безопасности, который ограничивает доступ веб-страницы к системе устройства.

  • 🔒 изоляция процессов
  • 🚫 запрет прямого доступа к ОС
  • 🧩 ограничение файловой системы
  • ⚙️ контроль выполнения скриптов

Цель Sandbox — предотвратить выполнение вредоносного кода и утечку данных.

🌐 Основные Sandbox-браузеры

🔵 Google Chrome

Один из самых строгих Sandbox-движков. Каждая вкладка работает как отдельный процесс.

👉 Intent поведение:

  • https:// → открытие в sandbox renderer
  • intent:// → передача в OS handler или блокировка
  • tg:// → внешний переход (если разрешено)

🟠 Microsoft Edge

Использует Chromium sandbox модель (аналог Chrome).

👉 Особенность:

  • усиленная интеграция с Windows security layer
  • ограничение deep link execution

🟣 Firefox

Использует собственную multi-process sandbox архитектуру (Gecko).

👉 Поведение:

  • меньше предсказуемости intent:// схем
  • часто блокирует нестандартные протоколы

🧬 Intent поведение в Sandbox средах

Схема Chrome Edge Firefox
https:// render render render
intent:// redirect / block OS handling blocked
tg:// external app external app fallback / block

🧪 Sandbox как Execution Layer

Sandbox браузеры — это не просто UI. Это execution isolation layer.

Они определяют:

  • как исполняются ссылки
  • может ли OS быть вызвана
  • разрешены ли intent-схемы

⚙️ Intent Portal интерпретация

В рамках Intent Portal Sandbox браузеры рассматриваются как:

🧠 ограниченные execution среды без прямого доступа к OS intent layer

Это означает:

  • нет полного контроля над intent://
  • поведение зависит от policy sandbox
  • execution всегда опосредованный

🧭 Вывод

Sandbox браузеры — это фундаментальный слой ограничения исполнения ссылок. Они формируют разницу между "открыть страницу" и "выполнить intent".
